# UIGen - AI-powered React Component Generator

## Project Overview
Next.js 15 app that lets users describe React components in a chat interface and see them generated live with preview. Uses Claude AI via Anthropic SDK and Vercel AI SDK.

## Tech Stack
- **Framework**: Next.js 15 (App Router, Turbopack)
- **Language**: TypeScript
- **UI**: React 19, Tailwind CSS v4, Radix UI, shadcn/ui
- **Database**: SQLite via Prisma (schema at `prisma/schema.prisma`)
- **AI**: Anthropic Claude via `@ai-sdk/anthropic` + Vercel AI SDK (`ai`)
- **Testing**: Vitest + Testing Library

## Setup
```bash
npm run setup   # install deps + prisma generate + prisma migrate dev
npm run dev     # start dev server (localhost:3000)
npm run test    # run tests with vitest
npm run db:reset  # reset database (destructive)
```

Set `ANTHROPIC_API_KEY` in `.env` to enable real AI generation (optional — static fallback exists without it).

## Key File Paths
```
src/
  app/
    page.tsx                     # Home page
    [projectId]/page.tsx         # Project view
    api/chat/route.ts            # AI chat API endpoint (streaming)
    main-content.tsx             # Main layout component
  actions/                       # Next.js Server Actions
    create-project.ts
    get-project.ts
    get-projects.ts
  components/
    chat/                        # Chat UI (ChatInterface, MessageList, MessageInput)
    editor/                      # Code editor (CodeEditor, FileTree)
    preview/PreviewFrame.tsx     # Live component preview
    auth/                        # Auth dialogs (SignIn/SignUp)
    ui/                          # shadcn/ui primitives
  lib/
    auth.ts                      # JWT-based auth logic
    prisma.ts                    # Prisma client singleton
    file-system.ts               # Virtual file system (in-memory)
    tools/file-manager.ts        # AI tool: create/update files
    tools/str-replace.ts         # AI tool: string replace in files
    prompts/generation.tsx       # System prompt for component generation
    transform/jsx-transformer.ts # Babel JSX → JS transform for preview
    contexts/
      chat-context.tsx           # Chat state context
      file-system-context.tsx    # Virtual FS state context
  middleware.ts                  # Auth middleware
prisma/schema.prisma             # DB schema (User, Project)
```

## Database Models
- **User**: id, email, password, projects[]
- **Project**: id, name, userId?, messages (JSON string), data (JSON string)

## Architecture Notes
- Virtual file system (no files written to disk) — files live in React context
- Components generated by Claude are transpiled client-side via Babel standalone
- Anonymous users can create projects (tracked via localStorage); sign-up to persist
- Auth uses JWT stored in HTTP-only cookies
- Prisma client generated to `src/generated/prisma/`
